## Dashboard Design Specification

### View 1 â€” Executive Summary (default landing view)

**Purpose:** One-screen answer to "Which customer segments should we prioritise?"

**Sheets to build:**

1. **KPI Banner** (Text / Summary cards)
   - Data source: `tableau_segment_kpi.csv`
   - Show: Total Revenue `ÂŁ17,073,078`, Total Customers `5,861`, Countries `41`, Unique Orders `36,639`
   - Use KPI card layout (big number + label)

2. **Revenue by Segment** (Horizontal Bar Chart)
   - Data source: `tableau_segment_kpi.csv`
   - Columns: `total_revenue` | Rows: `segment`
   - Colour by segment, sort descending by revenue
   - Add label: `pct_revenue`%
   - Key insight: Champions (25.2% of customers) â†’ 69.2% of revenue

3. **Segment Size vs Revenue Bubble Chart** (Scatter)
   - X: `pct_customers` | Y: `pct_revenue` | Size: `avg_monetary` | Colour: `segment`
   - Reference line at y = x (shows which segments punch above / below their weight)

4. **Interactive Filter:** Segment multi-select (applies to all sheets on the dashboard)

### View 3 â€” Geographic Distribution

**Purpose:** "Where are our customers and revenue concentrated?"

**Sheets to build:**

1. **World Map**
   - Data source: `tableau_country_summary.csv`
   - Geographic role: `country` â†’ Country/Region
   - Colour: `total_revenue` (continuous, green gradient)
   - Size: `customer_count`
   - Tooltip: country, total_revenue, customer_count, avg_order_value

2. **Top 10 Countries Bar Chart** (companion)
   - Filter: TOP 10 by `total_revenue`
   - Bars coloured by revenue rank

3. **Interactive Filter:** Minimum revenue threshold slider

---

### View 4 â€” Segment Drill-Down (Operational View)

**Purpose:** "Who exactly is in each segment â€” what do they look like?"

**Sheets to build:**

1. **RFM Scatter Plot**
   - Data source: `tableau_customer_detail.csv`
   - X: `recency` | Y: `monetary` | Colour: `segment` | Size: `frequency`
   - Log-scale Y axis (monetary is right-skewed)
   - Tooltip: customer_id, segment, recency, frequency, monetary, primary_country

2. **Segment Comparison Table** (Highlight Table / Heat Map)
   - Rows: `segment` | Columns: `avg_recency`, `avg_frequency`, `avg_monetary`, `pct_revenue`
   - Diverging colour: green = good, red = poor (direction per metric)

3. **Interactive Filters:**
   - Segment multi-select
   - Country filter
   - Monetary value range slider

## Calculated Fields to Create in Tableau

Add these calculated fields after connecting the data:

```
// Revenue per Customer
[total_revenue] / [customers]

// Customer Health Score (0â€“15 scale, higher = better)
[R_score] + [F_score] + [M_score]

// Segment Priority Label
IF [segment] = "Champions" THEN "1 - Retain"
ELSEIF [segment] = "Loyal Customers" THEN "2 - Reward"
ELSEIF [segment] = "Potential Loyalists" THEN "3 - Convert"
ELSEIF [segment] = "Big Spenders" THEN "4 - Upsell"
ELSEIF [segment] = "At Risk" THEN "5 - Win-Back"
ELSE "6 - Reactivate"
END

// Revenue Contribution Flag
IF [pct_revenue] >= 10 THEN "High Impact"
ELSEIF [pct_revenue] >= 5 THEN "Medium Impact"
ELSE "Low Impact"
END
```
